Welcome to the Fluid 3.0 demonstration

This mini-site demonstrates new features of Fluid 3.0.

About this site and source

This demo package serves two purposes:

  1. It can be rendered as a site (what you're viewing now, most likely)
  2. Its source files can be studied and contains plenty comments that explain how to use new features.

The site is therefore only part of the whole - studying the template files and index.php is equally recommended.

Inventory

The site is divided into dedicated sections that go in depth with one topic.

Vocabulary

Explains the new terms used to describe various parts of Fluid.

View the Vocabulary section.

Atoms

Atoms are a new feature in Fluid 3.0 which allows your template files to:

  • Become contained within a namespace, like ViewHelpers
  • Define parameters, like ViewHelpers
  • Embed various documentation and examples within the template file itself

View the Atoms feature section.

Expressions

Though also (partially) possible in Fluid 2.x and earlier, expressions have been improved in Fluid 3.0 and now allow any ViewHelper to also work as an expression. In Fluid, an expression is "inline syntax that is neither a variable or ViewHelper" - for example, {check ? thenVar : elseVar} is a ternary expression and is handled by the standard IfViewHelper.

View the Expressions feature section.

Parsing

Several new abilities have been added to the parsing layer of Fluid - this section describes some of the new features, explains up-coming breaking changes and the reason behind them, and provides alternative ways to achieve compatibility in templates.

View the Parsing feature section.

ViewHelpers

Explains new practices and API, new abilities and changes (breaking and otherwise) to the API used inside ViewHelpers. Explains how the compilation of templates is now removed and how ViewHelpers are rendered in a universal context - that there's no more difference between static and non-static rendering and how the ViewHelper instance always has access to child nodes, argument manipulation, node substitution and more.

View the ViewHelpers section.

PHP API

Details about the PHP side of Fluid's API - how to extract components from templates, how to render Fluid components (templates, sections, ViewHelpers) and how to use the new template registration functions.

View the PHP API section.

Documentation

A demonstration of how documentation embedded in Fluid templates or even sections can be rendered to generate a manual-type reference for all arguments, with examples/description.

View the documentation section.
